Relatório sobre Frameworks

O que são Frameworks e para que são utilizados?

Framework é um conjunto de bibliotecas, que abordam funcionalidades, e estruturas, para o desenvolvimento de aplicações, a fim de fornecer soluções para um mesmo domínio de problema, permitindo a reutilização do seu código. Assim, através das diversas linguagens de programação, os Frameworks são criados e identificados pelas metodologias, propósitos, e implementações, aos quais os diversos tipos de aplicações, na maioria orientadas a objetos, poderão reutilizar suas estruturas e códigos.

Um Framework é formado por um conjunto classes implementadas em uma determinada linguagem de programação, que fornece recursos comuns já prontos, validados e testados, os quais podem ser usados para auxiliar o desenvolvimento de software, viabilizando maior eficiência na resolução dos problemas, otimização de recursos, e detecção de erros. Ele fornece um modelo de dados, usados para resolver um problema específico, abstraindo parte do código das soluções que estamos tratando.

Framework é um conjunto de classes e interfaces que impõe um modelo colaborativo e um padrão de interação ao qual o nosso programa deverá se adaptar, as dependências entre as classes são embutidas, com a comunicação entre os objetos já definida. Então o Framework possui a responsabilidade para definir o comportamento da aplicação e chamar cada método, ou seja é o Framework que chama o código da nossa aplicação, e é ele quem tem o controle sobre a nossa aplicação, definindo o seu comportamento e o fluxo de controle da aplicação, esse conceito é chamado de Inversão de Controle.

Liste 4 Frameworks utilizados para programação Web selecionados por você, um para cada linguagem (Python, Java, PHP e Javascript). Descreva estes frameworks segundo seus propósitos e diferencial.

Python: Flask

O principal objetivo do framework é prover um modelo simples para desenvolvimento web, ao mesmo tempo que tem a flexibilidade no uso da linguagem Python. Não possui camada de abstração de banco de dados, validação de formulários ou mesmo qualquer outro componente, no entanto bibliotecas de terceiros provêm essas funcionalidades. 


from flask import Flask
app = Flask(__name__)
@app.route("/")
def hello_world():
  return "Hello, World!"
  • from flask import Flask importa a classe Flask do módulo flask.
  • app = Flask(__name__) cria uma instância da classe Flask e a atribui à variável app.
  • __name__ é uma variável especial que representa o nome do módulo atual.
  • @app.route("/") mapeia a função hello_world() para a rota raiz ("/") da aplicação Flask.
  • def hello_world(): define a função hello_world(), que é a função de retorno de chamada para a rota raiz.
  • return "Hello, World!" é a resposta retornada pela função

Java: Spring

O Spring é um framework Java criado com o objetivo de facilitar o desenvolvimento de aplicações, explorando, para isso, os conceitos de Inversão de Controle e Injeção de Dependências. Dessa forma, ao adotá-lo, temos à nossa disposição uma tecnologia que nos fornece não apenas recursos necessários à grande parte das aplicações, como módulos para persistência de dados, integração, segurança, testes, desenvolvimento web, como também um conceito a seguir que nos permite criar soluções menos acopladas, mais coesas e, consequentemente, mais fáceis de compreender e manter. 


@SpringBootApplication
@RestController
public class DemoApplication {
  @GetMapping("/helloworld")
    public String hello() {
    return"Hello World!";
  }
}
  • @SpringBootApplication: Essa anotação é usada para marcar a classe como uma classe de inicialização do Spring Boot.
  • @RestController: Essa anotação é usada para marcar a classe como um controlador do Spring MVC que lida com solicitações HTTP e retorna respostas HTTP.
  • public class DemoApplication: Esta é a classe principal do aplicativo, que contém o método main para iniciar o aplicativo Spring Boot.
  • @GetMapping("/helloworld"): Essa anotação é usada para mapear o método hello() ao endpoint "/helloworld" com o método HTTP GET.
  • public String hello(): Este é o método que será executado quando o endpoint "/helloworld" for acessado.

PHP: Laravel

Laravel é um Framework PHP utilizado para o desenvolvimento web, que utiliza a arquitetura MVC e tem como principal característica ajudar a desenvolver aplicações seguras e performáticas de forma rápida, com código limpo e simples, já que ele incentiva o uso de boas práticas de programação e utiliza o padrão PSR-2 como guia para estilo de escrita do código. 


<?php
Route::get('/users/{user}', function (User $user) {
  return $user;
});
  • Route::get('/users/{user}', ...) define uma rota GET para o caminho "/users/{user}".
  • O {user} é um parâmetro de rota que pode receber um valor dinâmico. Ele será passado como argumento para a função de callback.
  • A função de callback é definida usando function (User $user) { ...} Ela recebe um objeto User como argumento.
  • Dentro da função de callback, o código simplesmente retorna o objeto User como resposta HTTP.

Javascript: React

React é um framework JavaScript criado pelo Facebook (atual Meta) que é usado para criar interfaces de usuário (UI) em aplicativos web. Ele é popular por ser fácil de usar, altamente flexível e escalável, e é usado por muitas empresas de tecnologia, incluindo o Facebook, Instagram e Airbnb. 


export default function MyApp() {
  return (
  <div>
  <h1>Welcome to my app</h1>
  <MyButton />
  </div>
  );
}
  • export default é uma instrução que exporta o componente de função como o valor padrão para que ele possa ser importado e usado em outros arquivos.
  • function MyApp() define uma função chamada MyApp, que é um componente React de função.
  • A função retorna JSX (JavaScript XML), que é uma sintaxe semelhante a HTML usada para definir a estrutura e o conteúdo do componente.
  • O JSX contém um elemento <div> que envolve um elemento <h1> com o texto "Welcome to my app" é um componente <MyButton />.
  • é uma instância de um componente chamado MyButton.
